Supervision réseau
avec Zabbix 7.4

Terminé M2L — Atelier de Professionnalisation mars 2026

Contexte & problématique

Dans le cadre de l'infrastructure simulée de la M2L (Maison des Ligues de Lorraine), l'objectif était de mettre en place une solution de supervision open source permettant de monitorer en temps réel l'état des serveurs et équipements réseau : disponibilité, charge CPU, mémoire, espace disque, services actifs. Réalisé seul sur VMware Fusion (macOS), infrastructure conforme à l'Annexe II.E du référentiel BTS SIO.

Travaux réalisés

  • 01Installation d'Ubuntu Server 24.04 LTS (minimized) sur VMware Fusion. Configuration réseau : IP 192.168.203.130, installation nano, accès SSH depuis le Mac hôte.
  • 02Ajout du dépôt officiel Zabbix 7.4 et installation des paquets : zabbix-server-mysql, zabbix-frontend-php, zabbix-apache-conf, zabbix-sql-scripts, zabbix-agent.
  • 03Installation et configuration MySQL 8 : création de la base zabbix + utilisateur dédié, import du schéma initial (server.sql.gz), activation temporaire de log_bin_trust_function_creators = 1.
  • 04Configuration de /etc/zabbix/zabbix_server.conf (DBPassword) et génération de la locale en_US.UTF-8 (absente sur Ubuntu minimized).
  • 05Démarrage et activation des services : zabbix-server, zabbix-agent, apache2.
  • 06Finalisation via l'assistant web sur http://192.168.203.130/zabbix. Connexion admin fonctionnelle, supervision du serveur Ubuntu local active.

Problèmes rencontrés & solutions

ProblèmeSolution
nano absent sur Ubuntu minimizedapt install nano -y
Chemin sql-scripts introuvableLocalisation via dpkg -L zabbix-sql-scripts
Locale en_US manquantelocale-gen en_US.UTF-8 + redémarrage Apache
Paquets apache-conf et sql-scripts non installésRelance apt install séparément

Commandes clés

apt install zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-sql-scripts zabbix-agent
mysql -u root -p -e "CREATE DATABASE zabbix CHARACTER SET utf8mb4;"
zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql -u zabbix -p zabbix
locale-gen en_US.UTF-8
systemctl enable --now zabbix-server zabbix-agent apache2
