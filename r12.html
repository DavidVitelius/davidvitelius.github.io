Incident critique 802.1X
/ RADIUS — MADIACOM

Terminé Digicel FWI — Client MADIACOM 17 avril 2026

Contexte & problématique

Intervention urgente chez MADIACOM (Guadeloupe et Martinique). Échec total des authentifications WiFi 802.1X sur 4 access points Meraki — taux de succès RADIUS à 0%. Environnement : Cisco Meraki + NPS Windows Server 2019 + PKI Active Directory. Cause racine identifiée : certificat CA racine expiré, entraînant une rupture complète de la chaîne de confiance EAP-PEAP.

Travaux réalisés

  • 01Détection de l'alerte 802.1X failure sur le dashboard Meraki (AP gp-madiacom-ap01, 10h15) — 96% Radius login failure, 4% EAPoL timeout.
  • 02Test de connectivité RADIUS depuis Meraki : 5/5 APs joignent le serveur RADIUS — problème applicatif confirmé, pas réseau.
  • 03Analyse des codes d'erreur : 96% Radius login failure, 4% EAPoL timeout — confirmation d'un problème d'authentification côté serveur.
  • 04Connexion RDP sur le serveur RADIUS → Event Viewer → Event ID 6273, Reason Code 262 : certificat CA expiré — cause racine identifiée.
  • 05Localisation de l'autorité de certification via certutil -config - -ping → VMADPRODC01.MADIACOM.local.
  • 06Renouvellement du certificat CA via certsrv.msc avec conservation de la clé privée existante.
  • 07Publication du nouveau certificat CA dans Active Directory : certutil -dspublish -f ... RootCA.
  • 08Mise à jour de la GPO NPS : gpupdate /force, renouvellement du certificat NPS via certlm.msc.
  • 09Redémarrage du service NPS : net stop ias puis net start ias.
  • 10Déploiement d'un SSID temporaire WPA2-PSK (MADIACOM-TEMP) pour permettre l'exécution de gpupdate sur les postes clients.
  • 11Vérification sur le dashboard Meraki : statut passe de Bad à Good, DHCP 100%, DNS 100% — authentifications 802.1X rétablies.
